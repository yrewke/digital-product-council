# Executive Post-Pricing Responses

## Owner / Buyer Response

- **Right:** Transparent setup/support/pass-through separation; fair defect treatment; milestone payments; recommending cheaper alternatives where appropriate.
- **Missed:** Each tier must state what restaurant responsibility or burden it removes.
- **Too high:** T0 managed AED 350 and T1 AED 7,500 standard where ongoing value is not visible.
- **Too low:** No owner-side price is obviously too low; T2 provider risk does not automatically create buyer value.
- **Unpriced:** Staff turnover, retraining, peak troubleshooting, missed-order response, vendor escalation, cancellation/export/offboarding.
- **Competitor issue:** Must answer why a POS/SaaS queue is not already enough.
- **Hosting issue:** Client-owned T2 +AED 1,500 plus AED 900/month feels punitive without a visible responsibility difference.
- **Dangerous module:** WhatsApp deterministic bot.
- **Required change:** Paid discovery for first T2s, no client-owned T2 launch cohort, responsibility matrix, discovery pricing for bots/payment/delivery/integrations.

## Operator / Hosting Response

- **Right:** Managed launch default, mandatory T2 support, finite allowances, defect/support separation, restaurant-operation exclusions.
- **Missed:** Named monitoring owner, shift handover, acknowledgement timeout, missed-order escalation, retraining, and fulfilment failure procedure.
- **Too high:** T0 managed AED 350 when no visible ongoing operational outcome exists.
- **Too low:** T2 AED 1,100/month if outage, restoration, vendor escalation, retraining, or peak rescue is included.
- **Unpriced:** Order-chasing, staff-error diagnosis, billing/DNS issues, cancellation/export/offboarding/collection.
- **Competitor issue:** Existing POS/SaaS may already provide one supported operational queue.
- **Hosting issue:** Migration AED 750-1,500 is valid only for a clean export; otherwise quote.
- **Dangerous module:** Telegram deterministic bot.
- **Required change:** Responsibility matrix and paid-discovery first T2 deployments.

## ROI / Cashflow Economist Response

- **Right:** Responsibility-first pricing, setup-only launch discounts, T2 support, milestone payments, alternative recommendations, defect treatment.
- **Missed:** Acquisition, account management, collections, retraining, rescue, incidents, vendor/security coordination, and offboarding.
- **Too high:** T0/T1 managed recurring without visible responsibility.
- **Too low:** T2 setup/monthly and AED 275 overage if interruption costs are high.
- **Competitor issue:** POS/SaaS/WhatsApp/aggregator already provides some jobs.
- **Hosting issue:** Client-owned may be underpriced; managed may imply too much.
- **Dangerous module:** WhatsApp deterministic bot.
- **Required change:** Every first-cohort T2 uses paid discovery; bots become custom; add responsibility matrix.

## Competitor / Cheap WordPress Response

- **Right:** Do not compete with cheap WordPress on price; recommend alternatives honestly.
- **Missed:** First-year managed totals require a visible responsibility advantage: T0 AED 7,800; T1 AED 15,300; T2 AED 28,200.
- **Too high:** T0 managed and T1 setup for simple-presence buyers.
- **Too low:** T2 recurring and client-owned setup adjustments if real incident responsibility remains.
- **Unpriced:** Qualification, renewal, billing failure, cancellation/export, retraining, missed-order escalation, vendor-policy disputes.
- **Competitor issue:** Must define when the package beats POS/SaaS, not merely acknowledge it.
- **Hosting issue:** Predictable monthly price versus incident-variable responsibility.
- **Dangerous module:** Any fixed deterministic bot price.
- **Required change:** Mandatory alternative-fit gate and remove fixed bot pricing.

## Corrections Accepted For Challenge Round

1. Require paid discovery for every first-cohort T2 before fixed implementation commitment.
2. Remove fixed or provisional deterministic-bot prices; classify them custom/discovery.
3. Require an alternative-fit gate before quoting.
4. Require a responsibility matrix covering monitoring, acknowledgement, outages, restoration, security, vendor failure, missed orders, migration, export, cancellation, offboarding, and after-hours work.
5. Client-owned T2 is unavailable during the initial launch cohort.
6. Migration is a clean-export allowance only; all broader transfer work is quoted.
