# Responsibility Ledger

| Responsibility | Restaurant | Provider standard scope | Third party / conditional |
|---|---|---|---|
| Menu accuracy, prices, availability, hours | Owns approval and routine updates | Initial setup within allowance; product defects | Managed updates separately scoped |
| Incoming order monitoring and acknowledgement | Owns per shift | Notification/workflow availability | Messaging-provider incidents excluded |
| Food preparation, quality, complaints, refunds | Owns | Software records where included | Gateway/provider rules apply |
| Pickup/manual delivery fulfilment | Owns | Configured choice/fields/statuses where included | Delivery partner owns contracted service |
| Payment settlement/disputes | Owns merchant account and decisions | Supported setup/workflow where included | Gateway charges/policies pass through |
| POS/delivery integration | Provides compatible accounts/access | Only explicitly supported adapter scope | Vendor access/fees/failures conditional |
| Hosting/API/message/AI/vendor costs | Pays/pass-through | Transparent configuration and budgets | Vendor pricing/policy changes |
| Support | Named business-hours product support | Defects and agreed allowance | After-hours/managed operations separate |
| Data export/cancellation/continuity | Accepts process | Defined export, backup, exit, and incident boundaries | Vendor limitations disclosed |
