# Anonymous Peer Review

## Review 1
- **Strongest insight:** Tier 0 scope and Tier 0 sellability are different claims.
- **Weakest assumption:** Existing demand automatically makes Tier 0 worth buying.
- **Feature wrongly included:** Source tracking may exceed tiny Tier 0 implementation unless kept minimal.
- **Feature wrongly delayed:** Simple WhatsApp handoff may be important enough for early sale.
- **Support risk missed:** Telegram monitoring failures.
- **Customer confusion risk:** Order capture may be mistaken for order management.
- **Correction:** Sell Tier 0 only with explicit monitoring/fallback boundary.

## Review 2
- **Strongest insight:** Tier 2 is the clearest differentiated operational package.
- **Weakest assumption:** Dashboard/status use will reduce leakage.
- **Feature wrongly included:** Manual delivery handling without documented process.
- **Feature wrongly delayed:** Availability controls are necessary with any meaningful volume.
- **Support risk missed:** Staff turnover and retraining.
- **Customer confusion risk:** Dashboard implies provider-managed operations.
- **Correction:** Require workflow sign-off and named shift owners.

## Review 3
- **Strongest insight:** Fixed Tier 3 disguises multiple responsibility models.
- **Weakest assumption:** Commerce modules can be combined safely.
- **Feature wrongly included:** Customer records by default.
- **Feature wrongly delayed:** Delivery rules can be valuable earlier when leakage is proven.
- **Support risk missed:** Refunds, disputes, privacy, vendor outages.
- **Customer confusion risk:** “Direct Commerce” sounds all-inclusive.
- **Correction:** Replace fixed Tier 3 with individually gated modules.

## Review 4
- **Strongest insight:** Hybrid balances buyer clarity and provider scope.
- **Weakest assumption:** Add-on catalog will stay finite.
- **Feature wrongly included:** Too many visible modules.
- **Feature wrongly delayed:** Catering enquiry can be a strong early module.
- **Support risk missed:** Qualification/quoting time.
- **Customer confusion risk:** Four tiers plus modules becomes feature pricing again.
- **Correction:** Show three base packages; reveal modules only after job diagnosis.

## Review 5
- **Strongest insight:** Validation should be attached to a real package.
- **Weakest assumption:** Buyers accept “validation” language.
- **Feature wrongly included:** Broad measurement/reporting in every launch.
- **Feature wrongly delayed:** Basic source/outcome logging is essential.
- **Support risk missed:** Analysis and reporting effort.
- **Customer confusion risk:** Validation sounds unfinished.
- **Correction:** Call it a time-bounded launch plan with fixed measurements and deliverables.
