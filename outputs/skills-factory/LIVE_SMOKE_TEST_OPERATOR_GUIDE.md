# Later Live Smoke Test Operator Guide

Do not run this during the skills-factory phase.

1. Confirm `nlm` and `notebooklm-mcp` resolve in the intended shell. Do not install another implementation.
2. Obtain human approval for the test and manually select the active authentication profile.
3. Create a run ID and set `per_run_budget=5` in the run notes.
4. Search local caches first and choose one bounded commercial question.
5. Draft the smallest notebook set and estimated fan-out cost before querying.
6. Record every attempted query in `working/ledgers/notebooklm-query-ledger.csv`.
7. Stop immediately after five chat queries, or earlier when the evidence pack is sufficient or a blocker appears.
8. Do not run deep research, import sources, change notebooks, log in, or switch profiles without separate explicit approval.
9. Produce an evidence pack and pause for human review before running the council.
