# Run 2B Handoff

Run 2B should use Run 2A as the economics gate.

## Use These Outputs

- ROI break-even thresholds by fulfilment model.
- Which fulfilment models are actually worth testing.
- Which restaurant segment is most likely to show positive ROI.
- Which channel stack fits best: website, WhatsApp bot, Telegram bot, or hybrid.
- Which fulfilment features increase burden without improving ROI.

## Do Not Re-decide

- Final tiers.
- Final public prices.
- Sales copy.
- Aggregator replacement claims.

## Run 2B Decision Question

Given the positive-ROI fulfilment models from Run 2A, what feature bundle and packaging structure best matches the economics without turning the product into an open-ended service?
